# 🛒 E-Commerce Database Testing System 

---

## 📘 Project Overview
This project focuses on **database testing** for an E-commerce system.\
It validates the database using **SQL queries** to ensure data accuracy,
integrity, and relationships.

---

## 🎯 Objectives
- Validate database schema structure  
- Perform CRUD operations testing  
- Ensure data integrity using constraints  
- Simulate real-world database testing scenarios  

---

## 🗂️ Database Schema

### Tables Included:
- Users  
- Categories  
- Products  
- Orders  
- Order_Items  
- Payments  
- Cart  

---

## 🧠 Table Description

| Table | Description |
|------|------------|
| Users | Stores customer details |
| Categories | Groups products into categories |
| Products | Stores product information |
| Orders | Stores order-level details |
| Order_Items | Stores product details per order |
| Payments | Stores payment information |
| Cart | Stores user cart items |

---

## 🧪 Testing Performed

### 1. Schema Testing
- Verified table existence  
- Checked column names, data types, constraints  

---

### 2. CRUD Testing
- Insert, Read, Update, Delete operations  
- Tested on Users table (applicable to all tables)  

---

### 3. Data Integrity Testing
- Primary Key validation  
- Foreign Key validation  
- NOT NULL constraints  
- CHECK constraints
